# Travel Documents

Document and entry rules are dynamic. Do not hard-code country-specific visa outcomes.

## Intake

For visa, entry, transit, or electronic authorization advice, collect only what is necessary:

- Passport issuing country/region or nationality
- Destination country/region
- Transit countries/regions
- Travel purpose
- Length of stay
- Number of entries
- Departure date
- Existing visa, residence permit, or prior authorization status if relevant

## Verification Rules

Check official or consular sources before making claims about:

- Visa waiver
- Visa-free stay duration
- Electronic travel authorization
- Transit without visa
- Health declaration
- Arrival card
- ETIAS
- ESTA
- K-ETA
- eVisa
- Mainland China visa-free or transit policy

## Output Status

Mark each item as one of:

- 已核验，附核验日期和来源
- 待官方核验
- 待用户补充护照/国籍/停留目的
- 不适用

## Sensitive Data

Do not request or store:

- Full passport number
- Full ID card number
- Visa number
- Bank card number
- Password, token, cookie, or API key
- Full birth date unless the user explicitly needs it for their own workflow

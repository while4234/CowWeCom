---
title: image-generation - 图像生成
description: 通过 Codex 登录凭证支持文生图、图生图和多图融合
---

`image-generation` 在 CowAgent 中负责排队、后台执行和回发图片。实际生图由
`scripts/generate.py` 直接复用本机 Codex 登录凭证完成。

## 运行模式

CowWechat 默认使用直接 Codex 凭证模式：

```json
"skill": {
  "image-generation": {
    "runtime": "codex_auth"
  }
}
```

当 `runtime=codex_auth` 时，`scripts/generate.py` 会读取本地 Codex auth JSON，
使用其中的 access token 和 account id 请求：

```text
https://chatgpt.com/backend-api/codex/responses
```

返回的图片会写入当前任务的 `output_dir`。这条路径不再使用 broker、不调用
Codex CLI、不走 `/images/generations`、`/images/edits`，也不会回退到第三方
中转 API。

默认 auth 位置是 `$CODEX_HOME/auth.json`，未设置 `CODEX_HOME` 时使用
`~/.codex/auth.json`。如需在部署中指定位置，可设置 `CODEX_AUTH_FILE`。不要把
token 写入聊天、配置文件或日志。

## 请求格式

`scripts/generate.py` 接收一个 JSON 参数：

```json
{
  "prompt": "把衣服改成红色，其他保持不变",
  "image_url": "C:\\path\\input.jpg",
  "quality": "medium",
  "size": "1K",
  "aspect_ratio": "1:1",
  "output_dir": "C:\\path\\job-output",
  "runtime": "codex_auth"
}
```

多图融合时，`image_url` 是路径数组。本地图片会被压缩到合适大小，并作为
`input_image` data URL 发送给 Codex backend。

成功时 stdout：

```json
{
  "model": "gpt-5.5",
  "images": [
    {"url": "C:\\path\\job-output\\result.png"}
  ]
}
```

失败时 stdout：

```json
{"error": "human-readable error"}
```

## 微信使用方式

- 单图图生图：用户先发送图片，再在缓存窗口内发送“把这张图里的衣服改成红色，其他不变”。
- 引用图片：用户长按图片回复“把背景换成办公室，人物不变”。
- 多图融合：用户连续发送多张图片，再发送“把这几张图融合成一张合照”。

微信通道会把下载后的图片写进文本引用：

```text
[图片: C:\path\input.jpg]
[image: C:\path\input.jpg]
```

如果请求看起来是图生图但没有找到输入图片，工具会提示用户先发送、回复或引用图片。

## 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `prompt` | string | 是 | 图像描述或编辑指令 |
| `image_url` | string / array | 否 | 图生图或融合用的图片路径/URL |
| `quality` | string | 否 | `low`、`medium`、`high` 或 `auto` |
| `size` | string | 否 | `1K`、`2K`、`4K` 或 `1024x1024` 这类像素值 |
| `aspect_ratio` | string | 否 | `1:1`、`3:2`、`2:3`、`16:9`、`9:16` |
| `runtime` | string | 否 | 使用 `codex_auth` 复用本机 Codex 登录凭证 |

## Codex 直接使用

当 Codex 自己在普通对话里使用本 skill 且内置 `image_gen` 可用时，应直接调用
`image_gen`。CowWechat 后台任务运行在 Codex 工具进程之外，所以使用
`codex_auth` 脚本路径。

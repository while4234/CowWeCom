# 高德通勤与旅行路线

`amap-cowwechat` 是 CowWechat 内置 Skill，使用高德地图 Web Service API 查询通勤时间、实时路况、路线 ETA、POI、地理编码，并为旅游路线分析交通方式和行程合理性。

## 创建 Key

在高德开放平台创建应用：

1. 新建应用时，应用类型建议选择“出行”。
2. 添加 Key 时，服务平台选择“Web服务”。
3. 当前 Windows/CowWechat 项目不需要 Android、iOS、微信小程序 Key。
4. 暂时不需要 Web 端 JS API Key；只有后续在浏览器或 WebView 中渲染地图路线时才需要额外申请。

## 配置环境变量

推荐配置：

```text
AMAP_WEBSERVICE_KEY=your-webservice-key
```

高级交通态势默认关闭。确认 Web Service Key 已开通对应权限后，可在本机开启：

```text
AMAP_ENABLE_ADVANCED_TRAFFIC=true
```

兼容别名：

```text
SKILL_AMAP_COWWECHAT_WEBSERVICE_KEY=your-webservice-key
AMAP_KEY=your-webservice-key
```

也可以通过 CowWechat 的环境变量工具配置：

```text
请帮我设置 AMAP_WEBSERVICE_KEY 为 <your-webservice-key>
```

不要提交 `.env`、`config.json` 或任何包含真实 Key 的文件。

## 高级交通态势

`AMAP_ENABLE_ADVANCED_TRAFFIC=true` 用于允许 CowWechat 在基础路线 `tmcs` 之外尝试高德高级交通态势能力。不开启时，路况仍会使用驾车路线规划返回的实时路况片段进行摘要，不影响通勤、路线 ETA 和旅游路线分析。

高级交通态势适合三类查询：

- 道路 road：查询明确道路、高架、环路或快速路，例如“五环路现在堵不堵”“查延安高架路况”。适合判断单条道路整体拥堵和主要拥堵段。
- 圆形 circle：查询某个地点周边指定半径，例如“公司附近 3 公里路况”“首都机场周边路况”。适合出发前查看周边路网压力。
- 矩形 rectangle：查询两个地点或一个城区范围覆盖的路网，例如“北京站到国贸范围内路况”“陆家嘴这片区域路况”。适合区域拥堵概览和绕行建议。

如果用户没有提供城市、半径或范围边界，先追问一个简短问题补齐信息。不要让用户在聊天中发送 Key，也不要在输出、日志或示例里展示真实 Key。

权限和额度降级规则：

- 高级交通态势接口权限不足、额度不足、限流或当前区域不支持时，应自动降级到基础路线规划和 `tmcs` 路况摘要。
- 降级后仍应返回可用的通勤、路线、ETA 或旅游路线结果，可简短说明“高级路况不可用，已使用路线实时路况估算”。
- 不要为了高级路况反复重试消耗配额；一次请求内只做必要查询。

CLI 包装器示例：

```powershell
python "<base_dir>\scripts\amap_cowwechat.py" traffic "北京南站" "故宫" --city 北京

$env:AMAP_ENABLE_ADVANCED_TRAFFIC='true'
python "<base_dir>\scripts\amap_cowwechat.py" traffic-road "东三环" --adcode 110000
python "<base_dir>\scripts\amap_cowwechat.py" traffic-circle "116.305776,39.986414" --radius 3000
python "<base_dir>\scripts\amap_cowwechat.py" traffic-rectangle "116.351147,39.966309;116.357134,39.968727"
```

`traffic` 保持起终点路线 `tmcs` 路况查询；`traffic-road`、`traffic-circle` 和 `traffic-rectangle` 会调用高德高级交通态势接口，需要对应权限。

旅游路线会输出推荐顺序、每段交通方式、每段耗时、总交通耗时、合理性评分、风险提示和调整建议。

## 常见错误

- 未配置 Key：设置 `AMAP_WEBSERVICE_KEY`。
- Key 类型不对：请使用“Web服务”Key，不是 Android/iOS/JS SDK Key。
- 权限不足：在高德开放平台检查 Web Service API 权限。
- 地址解析失败：补充城市、区县、道路或门牌号。
- 额度不足或限流：检查高德控制台配额和调用频率。
- ETA 不稳定：实时路况会变化，最终时间以出发时查询为准。

## 安全说明

- 不要把真实 Key 写入源码、文档、日志、截图、Issue 或 Git 提交。
- 日志和回复只能显示脱敏信息，或完全不显示 Key。
- 如果 Key 已经在聊天或代码里暴露，建议在高德后台轮换。
- 如后续做 Web 地图展示，需要单独申请 Web 端 JS API Key，不能复用 Web Service Key。

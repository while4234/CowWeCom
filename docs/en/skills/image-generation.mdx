---
title: image-generation - Image Generation
description: Text-to-image, image-to-image, and multi-image fusion through direct Codex auth
---

`image-generation` starts image generation or editing jobs from CowAgent and
sends the result back to the original chat when the background job finishes.

## Runtime

CowWechat uses direct Codex-auth runtime:

```json
"skill": {
  "image-generation": {
    "runtime": "codex_auth"
  }
}
```

With `runtime=codex_auth`, `scripts/generate.py` reads the local Codex auth JSON,
calls `https://chatgpt.com/backend-api/codex/responses` with the logged-in Codex
access token and account id, then writes the returned image into the job output
directory. It does not use a broker, the Codex CLI, `/images/generations`,
`/images/edits`, or third-party image providers.

The default auth location is `$CODEX_HOME/auth.json`, falling back to
`~/.codex/auth.json`. Set `CODEX_AUTH_FILE` to override this path for a managed
deployment. Never paste token values into chat or config, and never log them.

## Request Shape

Run `scripts/generate.py` with one JSON argument:

```json
{
  "prompt": "turn the outfit red, keep everything else unchanged",
  "image_url": "C:\\path\\input.jpg",
  "quality": "medium",
  "size": "1K",
  "aspect_ratio": "1:1",
  "output_dir": "C:\\path\\job-output",
  "runtime": "codex_auth"
}
```

For multi-image fusion, `image_url` is an array of paths. Local images are
embedded as `input_image` data URLs for the Codex backend.

On success, stdout is:

```json
{
  "model": "gpt-5.5",
  "images": [
    {"url": "C:\\path\\job-output\\result.png"}
  ]
}
```

## WeChat Usage

- Send an image, then within the cache window send an edit instruction such as
  "make the outfit red and keep everything else unchanged".
- Reply to or quote an image with an edit instruction such as "replace the
  background with an office, keep the person unchanged".
- Send multiple images, then ask to combine them. The tool passes all image
  paths as an `image_url` array.

CowWechat represents downloaded image references in text as:

```text
[图片: C:\path\input.jpg]
[image: C:\path\input.jpg]
```

If an image editing request has no input image, the tool asks the user to send,
reply to, or quote an image first.

## Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `prompt` | string | Yes | Image description or edit instruction |
| `image_url` | string / array | No | Input image path(s) or URL(s) for editing or fusion |
| `quality` | string | No | `low`, `medium`, `high`, or `auto` |
| `size` | string | No | `1K`, `2K`, `4K`, or pixel value such as `1024x1024` |
| `aspect_ratio` | string | No | `1:1`, `3:2`, `2:3`, `16:9`, `9:16` |
| `runtime` | string | No | Use `codex_auth` for direct logged-in Codex credential runtime |

## Codex Direct Use

When Codex itself uses this skill inside a normal Codex chat and the built-in
`image_gen` tool is available, call `image_gen` directly. CowWechat background
jobs use the direct `codex_auth` script path because they run outside the
in-process Codex tool environment.
